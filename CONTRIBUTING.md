# Contributing to sanapo-net

We welcome any contributions to improve sanapo-net! Here’s how you can help.

## Licensing of Contributions
By submitting a Pull Request, you agree that your contribution will be licensed under the same license as the project (Business Source License 1.1 until 2029-01-01, then Apache 2.0 License).


## How to contribute

1. **Fork the repository** and create a branch for your feature/fix (`git checkout -b feature/AmazingFeature`).
2. **Commit your changes** (`git commit -m 'Add some AmazingFeature'`).
3. **Push the branch** (`git push origin feature/AmazingFeature`).
4. **Open a Pull Request**.

## Areas needing help


* **GUI upgrade**: Help migrate from Tkinter to PyQt6 or another fast, popular framework.
* **Topology engine**: Optimize rendering for 2000+ nodes.
* **Scanning**: Add new scan types (beyond ICMP).
* **Database**: Implement additional DB backends (PostgreSQL, MySQL).
* **Testing**: Write unit and integration tests.
* **Documentation**: Improve README, ARCHITECTURE, and user guides.
* **Performance**: Optimize multi‑threaded scanning and data buffering.


## Code standards

* Follow the project’s coding style.
* Add clear comments for non‑trivial code.
* Ensure all tests pass before submitting a PR.
* Use descriptive commit messages.

## Communication

* Open an **Issue** for ideas, questions, or bug reports.
* Contact the author directly: Alexander Vasilievich Polykov (SanaPo).

Thank you for helping make sanapo-net better!
